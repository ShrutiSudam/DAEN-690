Prioritized User List

Prioritized Users list provides information about which users need training on priority and the area in which they need training. This is the list of all users making erroneous transactions, sorted based on their weighted score. The weighted score indicates the amount of adverse impact a user can have on the system; the higher the weighted score, the more the adverse impact the user has on the system. The columns from REC_CREATION to MORE_1_MODULE help to show the area in which a user is making an error in turn the area in which he/she needs training. Value of 1 or 0.5 indicates errors in those transactions and 0 indicates correct transactions.


Column names and description:

  • USER_ID: ID of the user making erroneous transactions.
  • WEIGHTED_SCORE: Weighted score of a user.
  • REC_CREATION: how an asset was received into the system - 0 indicates purchase request through the system, 1 indicates manual entry. REC_CLASS: FAP classification – 0 indicates correct classification, 1 indicates incorrect classification.
  • LOC_DOC: Location movement document – 0 indicates valid system entry, 1 indicates invalid entry.
  • DISP_DOC: Disposal document – 0 indicates valid document present, 1 indicates it is absent.
  • DISP_RET_YR: Disposal retirement year – 0 indicates correct entry, 1 indicates incorrect entry.
  • DISP_SCAN_MIS: Scan type while disposing an asset – 0 indicates scanned, 1 indicates missing.
  • DISP_SCAN_MAN: Scan type while disposing an asset – 0 indicates scanned, 0.5 indicates manual.
  • MORE_1_MODULE: Error in more than one module – 0 indicates no, 1 indicates yes.
